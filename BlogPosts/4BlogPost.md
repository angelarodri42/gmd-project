The milestone 2 was supposed to focus on refining the gameplay. However, I ended up switching parts with milestone 3 because of bugs I could not figure out. Very professional project management, obviously.
Since the last milestone, the 1st thing I added was pickups. They were fairly simple because they worked similarly to obstacles. So, I created a ChocolatePickUp script with movement speed and a destroy method when collected.
After adding pickups, the old overlapping spawn issue became much worse. I spent DAYS trying to fix it because I wanted the spawning system to work before adding enemies. I tried combining all spawner scripts into one, adding cooldowns after each spawned object, and writing a long if statement to check for existing objects. Nothing worked, and I was slowly losing my mind. Eventually, I realised the issue was not the spawner logic. Each prefab had a child object with a different local X offset. So even though the spawner placed the prefab root correctly, the visible sprite appeared shifted. What a silly mistake.
Because of that whole adventure, I moved enemy implementation to milestone 3. The spawning issue was fixed, but I did not want to immediately break it again by adding more moving objects. So I moved on to something calmer.
I then improved the character animation. The running animation was already working, so I added the jumping animation and connected it to the Player script using a boolean that checks whether the character is jumping. I also adjusted the gravity and jump force.
After that, I worked on the chocolate bar, which tracks how much chocolate the player has collected. I made a ChocolateBarUI script that reads the chocolate percentage from the ChocolateManager. The ChocolateManager fills the bar when chocolate is collected, drains it over time, and checks whether it is above 50%, which will later allow the character to attack. This part was tedious, but not too difficult.
I also added more UI. Jakob’s PowerPoint mentioned that the game should have a quit option, so I added that together with a Game Over panel. The Game Over panel has buttons for going back to the main menu or quitting the game. I also made a pause menu with resume, main menu, and quit options. Finally, I added a main menu with play and tutorial buttons, although the tutorial button doesnt do anything yet. Future me problem.
Since I now had a Game Over panel, I could finally implement losing. At first, I made every obstacle kill the player instantly, but that made the game feel boring. I liked that the player could jump on bushes to reach platforms, so I changed the logic. Now, only tree stumps and enemies will kill the player. Falling too far behind will also make the player lose.
The last thing I added was a score system, because there was no way to track how long the player had survived. The score increases by 10 meters every time an old ground piece is moved forward. It is simple, but it works well enough.
